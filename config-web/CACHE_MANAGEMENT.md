# 缓存管理功能使用说明

## 功能概述

配置中心前端已成功集成了完整的缓存管理功能，支持对 Redis 缓存进行监控和操作。

## 新增功能

### 1. 配置列表页面增强

- ✅ 添加了缓存管理下拉菜单
- ✅ 显示系统状态和 Redis 状态
- ✅ 支持刷新当前环境缓存
- ✅ 支持刷新所有环境缓存
- ✅ 实时显示缓存操作状态

### 2. 专门的缓存管理页面

- ✅ 系统状态概览（系统状态、Redis 状态、缓存策略、最后刷新时间）
- ✅ Redis 详细信息展示
- ✅ 缓存操作面板（支持按环境刷新）
- ✅ 缓存信息说明
- ✅ 缓存列表展示
- ✅ 操作日志记录

### 3. API 接口增强

- ✅ `POST /api/v1/health/cache/refresh/{environment}` - 刷新指定环境缓存
- ✅ `POST /api/v1/health/cache/refresh` - 刷新所有环境缓存
- ✅ 完整的 TypeScript 类型定义
- ✅ 错误处理和状态管理

## 使用方法

### 在配置列表页面

1. 点击右上角的"缓存管理"下拉菜单
2. 查看 Redis 状态和缓存策略
3. 选择"刷新当前环境缓存"或"刷新所有环境缓存"
4. 等待操作完成，查看结果提示

### 在缓存管理页面

1. 访问 `/cache` 路由进入缓存管理页面
2. 查看系统状态概览
3. 查看 Redis 详细信息
4. 使用缓存操作按钮进行各种缓存刷新操作
5. 查看缓存列表和操作日志

## 技术实现

### 后端增强

- **CacheService.java**: 添加了 `clearAllConfigCache()` 和 `clearEnvironmentCache()` 方法
- **ConfigService.java**: 添加了 `refreshEnvironmentCache()` 和 `refreshAllCache()` 方法
- **HealthController.java**: 添加了缓存刷新 REST 接口
- **RedisConfig.java**: 优化了 LocalDateTime 序列化处理

### 前端增强

- **config.ts**: 添加了缓存管理相关的 API 接口和类型定义
- **Config/index.tsx**: 集成了缓存管理下拉菜单和状态显示
- **Cache/index.tsx**: 创建了专门的缓存管理页面
- **.umirc.ts**: 添加了缓存管理页面路由

## 缓存策略

- **三级缓存策略**: 本地缓存 → Redis 缓存 → 数据库
- **缓存过期时间**: Redis 缓存 1 小时，本地缓存 5 分钟
- **缓存键格式**: `config:配置键:环境`
- **自动降级**: Redis 不可用时自动降级到数据库

## 注意事项

1. **生产环境操作**: 在生产环境进行缓存刷新时请谨慎操作，建议在低峰期进行
2. **缓存刷新时间**: 缓存刷新操作可能需要一些时间，请耐心等待
3. **错误处理**: 系统会自动处理 Redis 连接异常，并降级到数据库
4. **状态监控**: 建议定期检查 Redis 状态，确保缓存服务正常运行
