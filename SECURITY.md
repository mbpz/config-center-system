# 安全政策

## 支持的版本

| 版本 | 支持状态 |
|------|---------|
| v0.1.x (开发中) | ⚠️ 不接受安全报告（仍在快速迭代） |
| v1.0.0+ (未来) | ✅ 支持 |

> ⚠️ 当前项目处于 v0.0.0 开发阶段，尚未发布首个稳定版本。
> 对于生产使用，请等待 v1.0.0 发布。

## 报告安全漏洞

**请勿通过公开 Issue 报告安全漏洞。**

### 私下报告

请发送邮件至: **apples398@163.com**

邮件请包含:
- 漏洞类型和影响范围
- 复现步骤或 PoC
- 建议的修复方案（如有）
- 你的联系方式（可选，用于更新状态）

### 处理流程

1. **确认收到**: 2 个工作日内确认
2. **调查评估**: 7 个工作日内确认漏洞并评估严重性
3. **修复发布**: 根据严重性，在 7-30 天内发布修复版本
4. **公开披露**: 修复后公开致谢（除非你希望匿名）

## 安全最佳实践

如果你在生产环境部署 Config Center，请确保:

- [ ] 修改所有默认密码 (CC_ADMIN_PASSWORD, CC_USER_PASSWORD)
- [ ] 设置强加密密钥 (CC_MASTER_KEY, 16+ 字符)
- [ ] 数据库和 Redis 启用 SSL/TLS
- [ ] 不在版本控制中提交 .env 文件
- [ ] 定期轮换密钥和密码
- [ ] 限制 /api/v1/configs 端点的访问（需要 ADMIN）
- [ ] 启用审计日志追踪配置变更
- [ ] 在 K8s 网络策略中限制 Pod 间通信

## 已知限制

当前版本 (v0.1.0) 的安全限制:

1. **用户存储**: 使用内存用户存储，不支持外部 LDAP/OAuth (Phase 2)
2. **密钥管理**: 单一主密钥，不支持 per-tenant 密钥轮换 (Phase 2)
3. **会话**: httpBasic + 简单 formLogin，无 JWT/OAuth (Phase 2)
4. **速率限制**: 未实现登录速率限制 (Phase 2)
